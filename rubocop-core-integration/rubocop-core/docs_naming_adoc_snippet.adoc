== Naming/MethodNameGetPrefix

[source,ruby]
----
# bad
def get_user(id)
  SpecializedUserClass.find(id)
end

def get_db_line_item(order_id, line_item_id)
  # ...
end

# good
def user_for(id)
  SpecializedUserClass.find(id)
end

def db_line_item_for(order_id, line_item_id)
  # ...
end

# Also good - using find_ pattern
def find_user(id)
  SpecializedUserClass.find(id)
end
----

Flags methods that start with `get_` prefix and take arguments, suggesting
more idiomatic Ruby naming conventions.

The cop automatically excludes:
* Accessor methods (methods without arguments) - handled by `Naming/AccessorMethodName`
* Methods that make HTTP GET requests (e.g., `connection.get`, `HTTP.get`, `Net::HTTP::Get.new`)
* API client methods in files matching patterns like `*client*.rb`, `*controller*.rb`, etc. that call `get()`
